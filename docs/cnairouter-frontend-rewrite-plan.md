# cnairouter 前端全面重构方案

## 1. 目标

把当前前端从“功能集合”升级为“有品牌叙事的产品平台 + 高效率控制台”。

核心结果：
- 对外像 OpenRouter 一样清晰传达价值：统一入口、模型目录、价格透明、稳定性、企业能力。
- 对内保留管理台效率：频道、密钥、用户、账单、日志、系统设置都能快速操作。
- 统一视觉、信息架构、组件和交互，避免首页、价格页、后台像三个不同项目。

## 2. 参考 OpenRouter 的可借鉴点

- 首页首屏直接给出一句产品定位，并配合核心指标建立信任。
- 模型/路由页强调“比较、筛选、排序、查看细节”，把决策链路压缩到一屏内。
- 定价页按 Free / Pay-as-you-go / Enterprise 分层，突出透明计费、无加价、支持企业采购。
- 企业页强调零留存、自动故障切换、观测、配额、SLA、合规和付款方式。
- About 页不是“公司介绍”，而是“规模、增长、定位、可信度”的证明页。
- 页面视觉统一使用克制的渐变背景、卡片网格和数据化排版。
- 顶部导航极简，但信息密度高，首页直接承载模型、排行、企业、价格、文档等入口。
- 首页先讲价值，再讲数据，再讲产品能力，最后给 CTA。
- 模型页强调搜索、筛选、对比、详情抽屉，而不是只做列表。
- 定价页用清晰的套餐对比 + FAQ 建立信任。
- 排行榜和公告页负责“持续更新感”，让产品看起来活跃且可信。
- 整体视觉是“克制、数据驱动、强可信”，不是普通后台风格。

## 3. 当前前端的重构定位

- `home`：对外品牌首页
- `pricing`：模型与价格目录
- `rankings`：模型/供应商趋势页
- `about` / `docs` / `enterprise`：信任与转化页
- `_authenticated/*`：高效率控制台

## 4. 现状问题

- 公开页和后台页的视觉语言不完全统一。
- 首页当前偏通用模板，产品定位不够强。
- 模型/价格页已有信息密度，但层级、叙事和目录感还不够像“产品目录”。
- 控制台页大多是功能型表格，缺少统一的页面骨架和节奏。
- 导航与入口层级较弱，不利于把“浏览模型 -> 了解价格 -> 注册 -> 获取 Key”串成闭环。

## 5. 新的信息架构

建议主导航调整为：
- 模型库
- 路由/融合
- Playground
- 排行榜
- 应用案例
- 企业方案
- 定价
- 文档
- 登录 / 进入控制台

建议页面职责：
- 首页负责“为什么选 cnairouter”
- 模型页负责“有什么模型、多少钱、什么能力”
- 排行榜负责“谁在涨、谁在跌、谁更稳定”
- 企业页负责“安全、SLA、合规、结算”
- 控制台负责“开通、配置、监控、分析”

## 6. 视觉系统

- 主基调：深石墨 / 冷白，辅色用青蓝或电蓝，价格与提醒用琥珀色。
- 版式：大标题 + 强数字 + 卡片网格 + 轻微渐变背景。
- 组件语言：圆角更克制，边框更细，阴影更浅，层级靠留白和对比而不是重装饰。
- 动效：页面淡入、卡片错位出现、抽屉滑入、数据切换过渡。
- 目标：既像产品官网，也像专业数据平台。

## 7. 页面级改造

### 首页

- 首屏用一句话说清楚 cnairouter 的价值。
- 加入 3-4 个强数字指标，如模型数、供应商数、可用率、请求量。
- 增加“快速开始”双 CTA：获取 API Key / 浏览模型。
- 用 3 个核心能力模块替代传统功能堆砌：统一接口、稳定路由、价格透明。
- 增加 Featured Apps / Featured Models / Recent Announcements。

### 模型与价格页

- 顶部保留搜索，但把筛选升级为“左侧筛选栏 + 顶部工具条 + 右侧内容区”。
- 默认提供卡片/表格双视图。
- 每个模型卡片展示：价格、上下文、模态、延迟、可用供应商、标签。
- 点击后进入详情抽屉，包含价格、能力、接口参数、适用场景、稳定性、示例。
- 保留现有过滤逻辑，但把 UI 从“表格式数据页”升级为“目录型产品页”。

### 排行榜页

- 用时间周期切换做主线：今日 / 周 / 月 / 年 / 全部。
- 强化趋势卡：上升模型、下降模型、市场份额、热度变化。
- 将图表和摘要并排展示，减少用户在不同区域切换的成本。

### 企业与关于页

- 重点展示：数据政策、SLA、付款方式、权限控制、团队协作、风控能力。
- 内容结构参考 OpenRouter 的 Pricing/Enterprise 叙事，但表达要贴合 cnairouter。

### 控制台

- 保持左侧导航 + 顶部操作区 + 内容区的稳定结构。
- 所有后台页面统一 `SectionPageLayout` 语义。
- 表格页强调批量操作、搜索、筛选、导出、空状态引导。
- 仪表盘页强调汇总指标、时间范围、趋势图和 drill-down。

## 8. 组件体系重构

建议建立 4 组共享组件：
- `shell`：导航、头部、侧边栏、页脚、公告条
- `marketing`：Hero、Metrics、Feature Grid、Social Proof、CTA、FAQ
- `catalog`：Search Bar、Filter Rail、Model Card、Compare Drawer、Details Drawer
- `analytics`：Stat Card、Trend Chart、Rank List、Snapshot Panel

优先复用现有基础组件，不重写底层 UI，只重组页面结构和信息层级。

## 9. 数据与状态

- 首页、模型页、排行榜页共用一套稳定的缓存策略。
- 目录页与控制台页分开 queryKey，避免互相污染。
- 重组件页面统一懒加载，图表和详情抽屉按需加载。
- 预取关键数据：首页进入时预取模型目录，排行榜页预取当前周期快照。

## 10. 国际化与可访问性

- 所有文案继续走 i18n。
- 公开页和控制台页都要保证键盘可操作、焦点可见、语义标签完整。
- 数字、价格、单位、时间范围必须统一格式化。

## 11. 实施路线

### 第一阶段

- 统一设计 tokens、间距、标题层级和颜色。
- 重构公共导航、页头、页脚、公告条。
- 完成首页新骨架。

### 第二阶段

- 重构模型与价格页。
- 重构排行榜页。
- 统一公开页视觉语言。

### 第三阶段

- 重构控制台壳层。
- 统一后台页的标题区、动作区、内容区。
- 处理表格、抽屉、空状态、错误态。

### 第四阶段

- 优化性能、动画、可访问性和响应式细节。
- 补齐埋点、SEO、文案和 i18n。

## 12. 成功标准

- 首页能在 5 秒内讲清楚产品价值。
- 模型目录支持“查找 - 对比 - 决策”完整闭环。
- 控制台页面风格统一，不再出现页面碎片化。
- 公共页的转化链路更短：浏览 -> 理解 -> 注册/登录 -> 获取 API Key。
